Composables (Shared State)

  • useFilters Singleton — holds the 4 global filter refs (period, location, category, status). All views read from and react to this single instance.
  • useI18n Locale persistence (localStorage). Switches translations EN↔JA. Drives currency (USD @ $1 / JPY @ ¥150).
  • useAuth Mock user session, role, avatar. Provides task data (demo only, no real auth).

Component Communication

  • FilterBar → Views FilterBar mutates useFilters singleton; all views are watchers — no props or events needed.
  • View → Modal View passes item via prop (:inventory-item). Modal emits @close to parent boolean flag.
  • App.vue → Modals App owns TasksModal and ProfileDetailsModal; shows them via boolean refs toggled by ProfileMenu events.

Key Design Decisions

  • No Pinia/Vuex Shared state managed via singleton composable pattern — simpler for demo scope.
  • No Database JSON files loaded into memory at startup. Fast, zero-setup, but not persistent.
  • SVG charts Custom SVG donut + bar charts — no charting library dependency.
  • Currency via i18n Language selection drives currency (EN=USD, JA=JPY) using a fixed ~150 exchange rate.
